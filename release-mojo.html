smart-release:release

Full name:

de.hilling.maven.release:smart-release-plugin:3-SNAPSHOT:release

Description:

Releases the project.

Attributes:

  • Requires a Maven project to be executed.
  • Executes as an aggregator plugin.
  • Executes by direct invocation only.

Optional Parameters

Name Type Since Description
bugfixRelease boolean - Perform a bugfix release. When performing a bugfix release, the last ("bugfix") number of the release version is incremented. The previous releases are determined from the .release-info.json-File.
Default value is: false.
User property is: bugfixRelease.
disableSshAgent boolean - (no description)
User property is: disableSshAgent.
knownHosts String - If set, this file will be used to specify the known_hosts. This will override any default value.
User property is: knownHosts.
modulesToForceRelease List - A module to force release on, even if no changes has been detected.
User property is: forceRelease.
noChangesAction NoChangesAction - Determines the action to take when no module changes are detected. Possible values: ReleaseAll, ReleaseNone, FailBuild
Default value is: ReleaseAll.
User property is: noChangesAction.
passphrase String - Specifies the passphrase to be used with the identityFile specified.
User property is: passphrase.
privateKey String - Specifies the private key to be used.
User property is: privateKey.
push boolean - Push changes to remote repository. This includes:
  • The newly created tag for this release containing the release information
  • The release info file containing the same information. This will be used to find relevant older releases to compare to during the following release.

Default value is: true.
User property is: push.
releaseGoals List -

The goals to run against the project during a release. By default this is "deploy" which means the release version of your artifact will be tested and deployed.

You can specify more goals and maven options. For example if you want to perform a clean, build a maven site, and then deploy it, use:


<releaseGoals>
    <releaseGoal>clean</releaseGoal>
    <releaseGoal>site</releaseGoal>
    <releaseGoal>deploy</releaseGoal>
</releaseGoals>

releaseProfiles List 1.0.1

Profiles to activate during the release.

Note that if any profiles are activated during the build using the `-P` or `--activate-profiles` will also be activated during release. This gives two options for running releases: either configure it in the plugin configuration, or activate profiles from the command line.
serverId String - If set, the identityFile and passphrase will be read from the Maven settings file.
User property is: serverId.
testBehaviour TestBehaviour - Determines running of tests. Possible values: testPhaseOnly, runAlways, skipTests
Default value is: testPhaseOnly.
User property is: testBehaviour.
testGoals List -

The goals to run against the project before the release. By default this is "test" which means the project is built and the tests are run.

You can specify more goals and maven options. For example if you want to perform a clean install, use:


<testGoals>
    <testGoals>clean</testGoals>
    <testGoals>install</testGoals>
</testGoals>

Remember that you will most probably do an implicit "compile package install deploy" during the release phase.
testProfiles List 1.0.1

Profiles to activate during the the test run.

Note that if any profiles are activated during the build using the `-P` or `--activate-profiles` will also be activated during test. This gives two options for running test: either configure it in the plugin configuration, or activate profiles from the command line.
